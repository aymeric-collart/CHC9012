Week 7: Presentation of PCIbex (Your first questionnaire)

CHC9012 | National Taiwan Normal University

Overview

This week, we will introduce the web-based platform PCIbex, which is used to design online questionnaires and experiments. By the end of this week, you will have the script of your very first experiment ready!

  • 🎯 Goals & Layout
  • 🧠 Key Concepts
  • 💻 Necessary Codes

What to expect from the platform:

  • Layout: What does this platform look like? Where do you type the script?
  • Assistance: How to access the documentation and the user forum.
  • Overall Presentation: Getting comfortable with the interface.
  • Basics: Setting up your first experiment from scratch.

Writing a simple yet powerful script:

  • Comments: How to comment a script on PCIbex to keep track of your structure.
  • Elements: How to create texts, tasks, and visual components.
  • Adjustments: How to adjust these elements to make them behave exactly as you want.
  • Logging: How to ensure the relevant results are stored correctly on PCIbex.

A brief summary of the essential elements:

  • newTrial(...): Used to introduce a trial in the experiment.
  • newText(...): Used to introduce a line of text.
  • newButton(...): Used to insert a button in the experiment.
  • newDropDown(...): Used to prompt the participants to select between several options.

1. Presentation of the Interface

Before we begin coding, you need to access the platform and set up your workspace.

Go to PCIbex (farm.pcibex.net)

⚠️ Important: Please don’t forget to verify your email after signing up, otherwise you will only be allowed to create one experiment!

Explore the different parts of the PCIbex interface below:

  • 1. Getting Started
  • 2. The Project Banner
  • 3. Folders & Files
  • 4. Coding Interface
  • 5. Actions Menu

The first thing to do is to create an account (click on “Sign up”). Once you have created your personal account, you can log in.

After logging in, you will see your dashboard. On the left side, you can find your projects. On the right side, you can find templates for specific paradigms.

Today, we will learn how to code a simple experiment from scratch. Let’s click on “Empty project”.

Once inside, you will see the main workspace.

Can you see where it is written Empty project_copy? Click on it to change its name. Good to know: The participants won’t see this name when you share the experiment, so you can be very explicit with your naming conventions!

This section contains two main parts:

  • Resources: So far, this is empty. This is where you upload files for your experiment (images, audio, or a .csv file where all your items can be found—we will talk about that next week!).
  • Scripts: There is already a file called main.js. For simple experiments, this is enough. This is the file we modify to code the whole experiment.

Once you open a script or an Excel-like file, you can modify it directly here.

The best feature is that you can test your experiment anytime after modifying it just by clicking “Refresh”. (Be patient, it can take several seconds!)

The last part is the “Actions” menu, which gives you several options:

  • “Unpublished” button: Once your experiment is ready, click this before starting data collection! You can test your script as many times as you wish beforehand, and those test data will be stored separately.
  • Results: Where you can find the downloaded data of your experiment.
  • Share: Click this to obtain the public link to share your experiment online.

2. Let’s be organized: The structure of your experiment

2.1 Conceptual structure

Before you start typing any code, you must have a crystal-clear vision of the procedure for your experiment. Even though every experiment is unique, most follow a standard logical sequence:

The difference between experiments lies in the specific content of each of these blocks. You must be very explicit about what each section consists of before you begin.

✍️ Preparatory Activity: Map it out!

A highly effective way to visualize your overall procedure is to draw it by hand.

Sketch out a series of rectangles on a piece of paper, with each rectangle corresponding to a different step (Consent, Background, Instructions, etc.). Once you have drawn the whole experiment, your only remaining task is to “translate” those boxes into PCIbex code!


2.2 Create the structure on PCIbex: Comment lines

Now that we have our structure, let’s put it into PCIbex. Before writing the actual experiment commands, it is best practice to create a “skeleton” of your script using comments.

  1. Open the main.js file in the PCIbex coding interface.
  2. Delete all the default text except the very first line (PennController.ResetPrefix(null)).
  3. Build your outline using comments. In JavaScript, anything following two slashes // is ignored by the computer.

Here is what you need to add. (Notice the green lines indicating what you should type):

  PennController.ResetPrefix(null)

+ /// 1. Welcome page
+ // 1.1 Hello page, ICF
+ 
+ // 1.2 Background questionnaire
+ 
+ // 1.3 Instructions
+ 
+ /// 2. Practice trials
+ 
+ /// 3. Experimental trials
+ 
+ /// 4. Goodbye page
⚠️ Important Note!

In the examples on this website, the + sign simply highlights that a line is a new addition to the script. You should not actually type the + sign!

Once you finish, your clean, organized main.js script should look exactly like this:

PennController.ResetPrefix(null)

/// 1. Welcome page
// 1.1 Hello page, ICF

// 1.2 Background questionnaire

// 1.3 Instructions

/// 2. Practice trials

/// 3. Experimental trials

/// 4. Goodbye page

3. Creation of your first experiment

Now we will build the actual content of the experiment step-by-step. Before that, we will need some basics:

  • We will be taking about “functions”, which are also refered to as elements in PCIbex. An element comes with a name and parentheses, such as: newText(). The details to input in the parentheses.

  • PCIbex is a platform to run experiments, and therefore it requires trials. Therefore, we always need to start with the newTrial() element, even if it is not a trial in the proper sense.

📄 3.1 Welcome Page

  • A. Welcome Page
  • B. Background information
  • C. Instructions

Texts and buttons

The welcome page consists of paragraphs with: - Welcoming words, - The Informed Consent Form, - Buttons to continue to the next pages, - Another page to collect personal information.

Let’s start with the welcome page, and as mentioned above, with a newTrial() element. We want to give a name to this element, and then add text, as follows.

/// 1. Welcome page
// 1.1 Hello page, ICF

newTrial("Welcome page",
+ newText("This is the welcome page.")
+ ,
+ newText("You can type the Informed Consent Form here.")
)

When you run these lines, you will realize that it does not work at all. And this is normal, it is just because we did not specify what to do with all these elements! We need to add commands to these elements. What we want is that these texts are (a) displayed (i.e., printed) on the screen, and that it does not automatically run into the next step. Here is the updated version:

/// 1. Welcome page
// 1.1 Hello page, ICF

newTrial("Welcome page",
    newText("This is the welcome page.")
+     .print()
+     .center() //You can add more commands like that
    ,
    newText("You can type the Informed Consent Form here.")
+     .print()
+     .center()
+     .wait() //Please remark that this command is placed at the last element
)

We are running into a new problem. The procedure “waits” as we asked to, but how to go to the next page? We can solve this by adding a “Continue” button (using the newButton() element).

/// 1. Welcome page
// 1.1 Hello page, ICF

newTrial("Welcome page",
    newText("This is the welcome page.")
        .print()
        .center()
    ,
    newText("You can type the Informed Consent Form here.")
        .print()
        .center()
-        .wait()
+   ,
+    newButton("Continue")
+        .center()
+        .print()
+        .wait() //Please remark that we moved the 'wait' command here
)

And we are done with this first page!

Texts, text inputs and dropdown

Next, we turn to the page where we collect personal information, such as the age of the participants, the highest degree they obtained, the languages they speak, etc. We already know how to type some text, so let’s start with this:

// 1.2 Background questionnaire

newTrial("Background information",
    newText("We will ask you some questions about your background. Please feel free to skip some questions if you do not wish to answer.")
        .print()
        .center()
        .wait() 
    )

Now we need to add some text to prompt the participants to enter their age, and a box where they can directly write. We will proceed this way:

// 1.2 Background questionnaire

newTrial("Background information",
    newText("We will ask you some questions about your background. Please feel free to skip some questions if you do not wish to answer.")
        .print()
        .center()
-        .wait()
+    ,
+    newText("Your age:")
+        .print()
+        .center()
+    ,
+    newTextInput("Age") //Please not that we added "Age" just to refer back to this element later
+        .print()
+        .center()
+        .wait()
    )

Let’s say that we want to collect more data, but we want the participants to choose between different options. We will do that with the newDropDown() element, as follows:

// 1.2 Background questionnaire

newTrial("Background information",
    newText("We will ask you some questions about your background. Please feel free to skip some questions if you do not wish to answer.")
        .print()
        .center()
    ,
    newText("Your age:")
        .print()
        .center()
    ,
    newTextInput("Age")
        .print()
        .center()
-        .wait()
+    ,
+    newText("Your highest degree:")
+        .print()
+        .center()
+    ,
+    newDropDown("Degree", "Please select one option")
+        .add("Elementary school", "Junior high school", "Senior high school", "BA", "MA", "Ph.D.")
+        .print()
+        .center()
+        .wait()
    )

You can add as much as you want now that you understand the logic.

⚠️ Important! In the end, we will need to add a “Continue” button. If not, it will move to the next screen after the participants selected one option. Adding this button prevents such potential problems.

The story is not over yet, since we need to specify what information needs to be stored. We cover this point in Section 4.

Nothing new!

Now that you know how to add text, you just need to follow the same logic to add the instructions! Here is an example for this course:

// 1.3 Instructions

newTrial("Instructions",
    newText("This is the instruction page.")
        .print()
        .center()
    ,
    newText("In this experiment, you will ... [Please enter the instructions depending on your experiment].")
        .print()
        .center()
    ,
    newText("Example: In this experiment, you will read sentences on the screen. These sentences will appear one at a time. Your task is to judge these sentences [add a word about how to judge the sentences].")
        .print()
        .center()
    ,
    newText("Here is an example: 'I will must do it.' This sentence is not acceptable, please click on 'not acceptable'.")
        .print()
        .center()
    ,
    newText("Please click on 'I understood' if you read and understood the instructions.")
        .print()
        .center()
    ,
    newDropDown("InstructionsCheck", "Please select one option")
        .add("I read and understood the instructions.", "I did not read the instructions.")
        .print()
        .center()
    ,
    newButton("Continue")
        .center()
        .print()
        .wait()
    )

We just finished the first part of the experiment! The good news is that this is actually the most tedious coding part, so it will only get easier for the rest of the coding!

4.2 Details of the procedure and the aesthetics

Often, you will need to adjust the layout, sizing, or timing of your elements. Here is an example of how we can modify an existing trial to center text, resize images, and add new elements.

Task: 1. Center the text and make it unfold over time. 2. Resize the “fish-plural” image to 200x200 px. 3. Create a new image named “fish-singular” using the file 1fishSquareTank.png and resize it to 200x200 px.

// Experimental trial
newTrial("experimental-trial",
    newAudio("fish-audio", "2fishRoundTank.mp3")
        .play()
    ,
    newText("fish-sentence", "The fish swim in a tank which is perfectly round.")
+       .center()
+       .unfold(2676)
-       .print()
    ,
    newImage("fish-plural", "2fishRoundTank.png")
+       .size(200, 200)
        .print()
+   ,
+   newImage("fish-singular", "1fishSquareTank.png")
+       .size(200, 200)
+       .print()
    ,
    newKey("keypress", "FJ")
        .wait()
)
