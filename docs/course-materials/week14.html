A brief summary of this week’s new functions:

  • ggplot(): Initializes a ggplot object and creates the blank visual canvas, linking it to the specified dataset.

  • aes(): Generates aesthetic mappings, which connect specific columns/variables in the dataset to visual properties of the plot (like the x-axis, y-axis, color, or fill).

  • geom_boxplot(): Adds a box-and-whisker plot layer to visualize the distribution, median, and quartiles of continuous data.

  • labs(): Modifies labels across the plot, allowing you to customize the main title, subtitle, x-axis text, and y-axis text.

  • theme_minimal(): Applies a clean, minimalist design theme by removing the default heavy gray background while keeping faint gridlines.

  • theme(): Dynamically customizes deep aesthetic details of the active theme, used here to hide the chart legend (legend.position = “none”).

  • scale_y_continuous(): Controls the behavior, absolute limits, specific tick intervals (breaks), and out-of-bounds (oob) handling for a continuous y-axis.

  • 1:7: A shorthand colon operator that generates a sequential integer vector from 1 to 7 (1, 2, 3, 4, 5, 6, 7).

  • summarySE(): A helper function (typically from the Rmisc package) that automatically calculates descriptive summary statistics—including sample size (N), mean, standard deviation (sd), standard error (se), and confidence intervals (ci)—grouped by independent variables.

  • geom_col(): Draws a column chart where the heights of the bars represent the actual raw values provided in the y-axis aesthetic mapping.

  • geom_errorbar(): Adds vertical error bar elements to the plot using explicit upper (ymax) and lower (ymin) statistical boundaries.

  • geom_text(): Draws literal text labels directly onto the plot framework using specified positions and structural adjustments.

  • round(): Rounds numerical values to a specified number of decimal places (e.g., round(Value, 2) rounds to 2 decimal places).

  • geom_jitter(): Adds a scatter plot layer where data points are randomly shifted by tiny amounts (width and height) to prevent overlap and reveal data density.

  • seq(): Generates a highly controlled mathematical sequence of numbers defining a starting point, ending point, and uniform skipping step size (e.g., seq(0, 20000, by = 4000)).

  • %>%: The tidyverse pipe operator; passes the output data structure of the left-hand function directly as the first argument into the right-hand function.

  • distinct(): Retains only unique/distinct rows from an input dataframe based on specified column values, removing duplicate entries.

  • select(): Picks specific columns out of a large dataframe to keep, while dropping all unnamed columns.

  • factor(): Converts a standard character or numeric variable into a robust categorical Factor vector, allowing you to lock in custom chronological hierarchies via its levels argument.

  • c(): Combine/Concatenate function; groups standalone strings, variables, or elements into a unified vector.

  • geom_bar(): Draws a bar chart where the height of the bar is calculated automatically by counting the total number of observations falling into each category on the x-axis.

  • after_stat(): A specific ggplot evaluation function that instructs layers to calculate and map internal statistical steps (like count) after the initial data transformation is complete.

  • scale_fill_brewer(): Sets categorical color palettes using discrete color configurations originally designed by ColorBrewer.

  • theme_classic(): Applies a traditional presentation theme consisting of a pure white background with clean, high-contrast black x and y axis lines.

  • expansion(): Configures explicit numerical margins/padding along the outer edges of a scale, preventing text labels or bars from clipping at the boundaries.

👀 Part 1: Why using figures, and what kinds of figures?

Why plotting the data?

Strictly speaking, we can provide results tables, including all potentially interesting information, and even provide access to the full dataset as well. Then, why the need to visualize the data?

  • The nature of human cognition is such that figures are more attractive than tables and texts. It can therefore be very useful to communicate our message more effectively.
  • As we saw last week, summary figures can be very useful to understand the general pattern of results while acknowledging finer-grained variation.

Introduction to most commons types of figures in experimental linguistics:

  • 1.1 Box plots for data distribution
  • 1.2 Bar plots for categorical counts
  • 1.3 Line charts for continuous measurement, generally over time
  • 1.4 Scatter plots for relationships & correlations
  • 1.5 Combined plots for more complex data

  • What it does: Visualizes the spread, median, quartiles, and outliers of a continuous numerical variable across different categorical groups.

  • Core question: How does data distribution compare between groups? (e.g., Which Iris species has the longest petals?)

  • What it does: Displays counts or frequencies of distinct categories. By default, geom_bar() counts the number of rows for each category on the x-axis, hence the need sometimes to summarize the data beforehand.

  • Core question: Which categories are the most and least common in our dataset?

  • What it does: Connects individual data points chronologically or sequentially to reveal trajectories, spikes, or downward dips.

  • Core question: How is our metric moving over a continuous sequence?

  • What it does: Plots individual data points at the intersection of two continuous numeric variables to display positive, negative, or non-linear correlations.

  • Core question: Does variable X increase, decrease, or remain unchanged as variable Y changes?

Part 2: A step-by-step guide to plot figures with R

2.1 Introduction to ggplot and its spirit

Most researchers using R and even other programming languages often, if not always, use the ggplot2 package. It gives us full control of what we want to plot, we can customize the figure as we wish by adding legends, annotations, changing the theme and the colors, etc.

ggplot2 uses a philosophy called the Grammar of Graphics (Wilkinson, 2005). The core spirit of ggplot2 is that any data visualization can be broken down into a series of independent layers stacked on top of one another.

We know that there are many inter-item variation in this experiment. We can have a more complex figure to visualize all the information at the same time:

  • Bar plots for the general pattern,
  • Scatter plots to visualize the results from each trial.

We need two types of summary datasets: (1) general pattern, and (2) by-trial results.

JudgSummary <- summarySE(data_OtherPart_CleanAbsoluteRelative,
                         measurevar = "Value",
                         groupvars = "Condition")

JudgSummaryItem <- summarySE(data_OtherPart_CleanAbsoluteRelative,
                         measurevar = "Value",
                         groupvars = c("Condition", "ItemNo"))

Now we can plot the data. The geom_col() maps with the aes() related to the overall pattern. The geom_jitter() includes a second aes() that maps with the dataset including the by-trial data.

ggplot(JudgSummary,
       aes(x = Condition,
           y = Value,
           fill = Condition)) +
  geom_col(width = 0.5, alpha = 0.9) +
  geom_errorbar(aes(ymin = Value - ci,
                    ymax = Value + ci), 
                width = 0.15,
                size = 0.7) +
  geom_text(aes(label = round(Value, 2)), 
            vjust = 4,
            color = "white",
            size = 5,
            fontface = "bold") +
  scale_y_continuous(limits = c(1, 7),
                     breaks = 1:7,
                     oob = scales::rescale_none) +
  geom_jitter(data = JudgSummaryItem,
              aes(color = Condition),
              width = 0.2,
              height = 0.1,
              alpha = 0.3,
              size = 2.5)

It is the plot that we want, but it is not easy to see all the dots (1 dot = 1 trial). We can adjust the shades and colors, and it is done!

ggplot(JudgSummary,

2.3.3 Visualizing the background information

Our dataset includes more than 20 observations per participants, meaning that the same type of information (e.g., the age and highest degree of Participant 1) appears more than 20 times. Therefore, we need to create a new dataset where “1 row = 1 participant”.

ParticipantBackground <- data_OtherPart_CleanAbsoluteRelative %>%
  distinct(ID_1_TimeReceived, .keep_all = TRUE) %>%
  select(ID_1_TimeReceived, UserAge, UserDegree)

ParticipantBackground$UserDegree <- factor(ParticipantBackground$UserDegree, 
  levels = c("High school", "BA", "MA", "Ph.D.") )
  • Age of the participants
  • Highest degree of the participants
  • Plot “age” and “degree” at the same time
ggplot(ParticipantBackground, aes(x = "", y = UserAge)) +
  geom_boxplot(
    width = 0.4, 
    fill = "#4a90e2",
    color = "black",
    alpha = 0.7
  ) +
  theme_classic(base_size = 14) +
  labs(x = "",
       y = "Age") +
  theme(legend.position = "none")

It is easy to see that the majority of the participants are between 20 and 30 years old. There is only one participant who is much older.

ggplot(ParticipantBackground,
