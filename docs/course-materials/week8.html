Week 8 - PCIbex: Multiple-item questionnaires, Latin square and other paradigms

CHC9012 | National Taiwan Normal University

Overview

This week, we will discuss how to set the parameters of a template to obtain a more simple and yet more powerful script. By the end of this week, you will have the script of your very final experiment ready!

  • 🎯 Goals & key concepts
  • 💻 Necessary codes
  • 📝 Homework

How to optimize your script:

  • Template: How to create a script that can run automatically through all the trials?
  • Randomization: How to present the trials using a random order?
  • Counterbalancing: How to counterbalance the items using a Latin square design?

A brief summary of the essential elements:

  • Template(...): Used to introduce a trial template.
  • Sequence(...): Used to control the order of the different parts of the experiment.
  • randomize(...): Used to present the trials using a random order.
  • newVar(...): Used to create a new variable.
  • getX(...): Used to obtained the content given by the participants in a specific element (example: getTextInput(...), getDropDown(...), etc.)
  • getVar(...): Used to obtain the content of an element stored in a variable (i.e., within newVar(...)).

1. Let’s simplify and automatize the process

1.1 Creation of the template

So far, we created an experiment following this structure:

But more precisely, it looks like that, with multiple trials:

As we saw last week, we could type all the trials individually. Alternatively, there is a much more efficient way to go. We provide the details below.

  • A. Create the material table
  • B. Update the existing code

Instead of typing the trials in the script, we create a table where we can list them. Let’s say we have 8 sentences, divided between 2 conditions (so 4 sentences per condition)

ItemNo Sentence Condition
1 This is the first sentence Grammatical
2 This is the second sentence Grammatical
3 This is the third sentence Grammatical
4 This is the fourth sentence Grammatical
5 This are the fifth sentence Ungrammatical
6 This are the sixth sentence Ungrammatical
7 This are the seventh sentence Ungrammatical
8 This are the eighth sentence Ungrammatical

We need to have this tables under the Resources part on the left of the interface. One way to do is to open an Excel document, save it into a .csv file (columns seperated by commas), and upload on PCIbex. This is the most convenient way to go, especially when you have a lot of information.

Instead, we are going to create it directly for this tutorial. Right next to the Resources part, you can click on the + (Create new file) sign, and you rename it to: material.csv. You will see the logo changing to something similar to: .

After that, you click on the file, and it will open it next to the script. You just have to copy and paste the following section inside (the commas , indicate different columns).

ItemNo,Sentence,Condition
1,This is the first sentence,Grammatical
2,This is the second sentence,Grammatical
3,This is the third sentence,Grammatical
4,This is the fourth sentence,Grammatical
5,This are the fifth sentence,Ungrammatical
6,This are the sixth sentence,Ungrammatical
7,This are the seventh sentence,Ungrammatical
8,This are the eighth sentence,Ungrammatical

Now that we have our materials uploaded onto PCIbex, we just need to modify the script that we already have. As a reminder, this is what it looked like with two trials:

Previous script 
/// 3. Experimental trials

newTrial("Experiment",
    newText("FirstSentence", "This is another grammatical sentence")
        .print()
        .center()
    ,
    newScale("Scale", "1", "2", "3", "4", "5", "6", "7")
        .labelsPosition("bottom")
        .before(newText("Not acceptable"))
        .after(newText("Acceptable"))
        .print()
        .center()
        .wait()
    )
    
newTrial("Experiment",
    newText("SecondSentence", "This sentence not is grammatical")
        .print()
        .center()
    ,
    newScale("Scale", "1", "2", "3", "4", "5", "6", "7")
        .labelsPosition("bottom")
        .before(newText("Not acceptable"))
        .after(newText("Acceptable"))
        .print()
        .center()
        .wait()
    )

We actually just need to update that a little bit… and you will realize that it will even look easier than what we wrote! More specifically:

  • We must specify that we are dealing with a template using the Template() function. The content of the template (i.e., the trials) is found within the parentheses.
  • We must also specify the name of the file where the trials can be found, we change the file into a variable to refer to (here, the name of the variable is row), and which column corresponds to the trials to display on the screen (here, the column is called Sentence; row.Sentence means that it is the column called Sentence in the file corresponding to the variable row).
  • Importantly, we delete all other newTrial() elements. We only need one within the template.
/// 3. Experimental trials

+ Template("material.csv", row => 
newTrial("Experiment",
-    newText("FirstSentence", "This is another grammatical sentence")
+    newText("Sentence", row.Sentence)
        .print()
        .center()
        .log()
    ,
    newScale("Scale", "1", "2", "3", "4", "5", "6", "7")
        .labelsPosition("bottom")
        .before(newText("Not acceptable"))
        .after(newText("Acceptable"))
        .print()
        .center()
        .wait()
        .log()
    )
+ ) // This is the closing parenthesis corresponding to `Template()`

- newTrial("Experiment",
-     newText("SecondSentence", "This sentence not is grammatical")
-         .print()
-         .center()
-     ,
-     newScale("Scale", "1", "2", "3", "4", "5", "6", "7")
-         .labelsPosition("bottom")
-         .before(newText("Not acceptable"))
-         .after(newText("Acceptable"))
-         .print()
-         .center()
-         .wait()
-    )

And that’s all, we are done!

1.2 log() with a template

If you run the experiment and look at the results file, you will see that you can find the judgments given by the participants. But we may want more information in the results file:

  • The ID of the sentence,
  • The sentence itself,
  • The experimental condition,
  • And other information found in the material table.
/// 3. Experimental trials

Template("material.csv", row => 
newTrial("Experiment",
    newText("Sentence", row.Sentence)
        .print()
        .center()
-        .log() // Since we know it does not work, let's delete it
    ,
    newScale("Scale", "1", "2", "3", "4", "5", "6", "7")
        .labelsPosition("bottom")
        .before(newText("Not acceptable"))
        .after(newText("Acceptable"))
        .print()
        .center()
        .wait()
        .log()
    )
+    .log("ItemNo", row.ItemNo)
+    .log("Sentence", row.Sentence)
+    .log("Condition", row.Condition)
)

In addition, since the participants provide background information, we would like to add these details in the same line as well! But such information is not found in a seperate file, they are provided by the participants themselves. This is therefore a two-step process to store such information in the results file, as follows:

  • Step 1 - Define the variable when the participants provide their information. We create a new variable (think as if we create a new column) called “Age” with the newVar() element. Then we set the content of this variable. We need to tell PCIbex where to find it with the getTextInput() element. In the same code, we have an element called newTextInput("Age"), therefore we need to use the same name: getTextInput("Age"). We need to do the same with other elements we want to keep, hence the getDropDown("Degree") element.
// 1.2 Background questionnaire

newTrial("Background information",
    newText("We will ask you some questions about your background. Please feel free to skip some questions if you do not wish to answer.")
        .print()
        .center()
    ,
    newText("Your age:")
        .print()
        .center()
    ,
    newTextInput("Age")
        .print()
        .center()
        .log("final")
    ,
    newText("Your highest degree:")
        .print()
        .center()
    ,
    newDropDown("Degree", "Please select one option")
        .add("Elementary school", "Junior high school", "Senior high school", "BA", "MA", "Ph.D.")
        .print()
        .center()
        .log()
    ,
    newButton("Continue")
        .center()
        .print()
        .wait()
+    ,
+    newVar("Age")
+        .global()
+        .set(getTextInput("Age"))
+    ,
+    newVar("Degree")
+        .global()
+        .set(getDropDown("Degree"))
    )
  • Step 2 - Store the information along with the results of each trial. We just need to add two lines, starting with the .log() command that we are very familiar with. We need to use the getVar() element to say “grasp the information from this variable”.
/// 3. Experimental trials

Template("material.csv", row => 
newTrial("Experiment",
    newText("Sentence", row.Sentence)
        .print()
        .center()
        .log()
    ,
    newScale("Scale", "1", "2", "3", "4", "5", "6", "7")
        .labelsPosition("bottom")
        .before(newText("Not acceptable"))
        .after(newText("Acceptable"))
        .print()
        .center()
        .wait()
        .log()
    )
    .log("ItemNo", row.ItemNo)
    .log("Condition", row.Condition)
+    .log("Age", getVar("Age"))
+    .log("Degree", getVar("Degree"))
)

2. Let’s specify the template and the overall procedure

We are almost done with the parameters of the experiment! Now that we are using the Template() element, we can set additional parameters.

  • A. Sequence and randomization
  • B. Counterbalancing

We can specify the sequence of the whole experiment right at the beginning of the script. In other words, we want to translate this figure into PCIbex language.

We can do that using the Sequence() element, as below:

Sequence("Welcome page", "Background information", "Instructions", "AnnouncementPractice", "Practice", "AnnouncementEndPractice", "Experiment", "Goodbye")

But we will run into a problem. We did not specify when to send the results! To do that, we must give a name to the SendResults() element that we placed right before the ‘goodbye page’, and include it in the Sequence() element.

PennController.ResetPrefix(null)

+ Sequence("Welcome page", "Background information", "Instructions", "AnnouncementPractice", "Practice", "AnnouncementEndPractice", "Experiment", "Send" "Goodbye")

/// 1. Welcome page
// 1.1 Hello page, ICF

newTrial("Welcome page",
...
    )

// 1.2 Background questionnaire

newTrial("Background information",
...
    )

// 1.3 Instructions

newTrial("Instructions",
...
    )

/// 2. Practice trials

newTrial("AnnouncementPractice",
...
    )

newTrial("Practice",
...
    )
    
newTrial("AnnouncementEndPractice",
...
)

/// 3. Experimental trials
Template("material.csv", row => 
newTrial("Experiment",
...
)

/// 4. Goodbye page
+ SendResults("Send")

newTrial("Goodbye",
...
        )

With the Sequence() element, we just need to specify which part of the experiment needs to be presented in random order, using the randomize() element, as below.

Sequence("Welcome page", "Background information", "Instructions", "AnnouncementPractice", "Practice", "AnnouncementEndPractice",
+ randomize("Experiment"), 
"Send", "Goodbye")

Our last step concerns Latin square designs, i.e., when we want to counterbalance the items and present different ones to different participants for some reasons. So far, we have 8 sentences dispatched into 2 conditions, so 4 items per condition. Now, let’s say that we only want to present the two first sentences from each condition to Participant 1, and the last two sentences to Participant 2, and so on.

With PCIbex, we can specify the lists directly in the material table. We just need to have a column called group, and PCIbex will automatically identify that these corresponds to different lists in a Latin square design. My habit is to use ‘A’, ‘B’, etc. to specify the lists, such that there is no confusion with the item IDs.

ItemNo Sentence Condition group
1 This is the first sentence Grammatical A
2 This is the second sentence Grammatical A
3 This is the third sentence Grammatical B
4 This is the fourth sentence Grammatical B
5 This are the fifth sentence Ungrammatical A
6 This are the sixth sentence Ungrammatical A
7 This are the seventh sentence Ungrammatical B
8 This are the eighth sentence Ungrammatical B

In the end, the content of material.csv looks like this:

ItemNo,Sentence,Condition,group
1,This is the first sentence,Grammatical,A
2,This is the second sentence,Grammatical,A
3,This is the third sentence,Grammatical,B
4,This is the fourth sentence,Grammatical,B
5,This are the fifth sentence,Ungrammatical,A
6,This are the sixth sentence,Ungrammatical,A
7,This are the seventh sentence,Ungrammatical,B
8,This are the eighth sentence,Ungrammatical,B

Finally, to keep track of which list the participants saw, we need to use the .log() command!

/// 3. Experimental trials
Template("material.csv", row => 
newTrial("Experiment",
    newText("Sentence", row.Sentence)
        .print()
        .center()
    ,
    newScale("Scale", "1", "2", "3", "4", "5", "6", "7")
        .labelsPosition("bottom")
        .before(newText("Not acceptable"))
        .after(newText("Acceptable"))
        .print()
        .center()
        .wait()
        .log()
    )
    .log("ItemNo", row.ItemNo)
    .log("Condition", row.Condition)
    .log("Sentence", row.Sentence)
+    .log("List", row.group)
    .log("Age", getVar("Age"))
    .log("Degree", getVar("Degree"))
)

3. Final test: Run the experiment several times and look at the results file

A good practice, when everything is set, is to run the experiments several times as if we were pretending that we are different participants. We can first check that this is indeed what we want the participants to see, and also that the results file is in order.
